Bike Sharing System - Data Analysis

Naming Strategy

Hash URIs for ontological terms, slash URIs for individuals

  • Ontological terms path: http://bikesharingsystem.linkeddata.org/bsh/ontology/BikeSharing#
  • Ontological terms example: http://bikesharingsystem.linkeddata.org/bsh/ontology/BikeSharing#hasSeason
  • Individuals path: http://bikesharingsystem.linkeddata.org/bsh/resource/
  • Individuals path: http://bikesharingsystem.linkeddata.org/bsh/resource/Register/2011-01-20_00:00:00

Column types:

  • Datetime -> object
  • Season -> int64
  • Holiday -> int64
  • Workingday -> int64
  • Weather -> int64
  • Temp -> float64
  • Atemp -> float64
  • Humidity -> int64
  • Windspeed -> float64

There are not null values in the dataset

Licensing choosed for our Dataset
The DBCL 1.0 is a license that is specifically designed for databases. It focuses on protecting the contents within a database rather than the structure of the database itself. The license permits users to freely use, modify, and distribute the database contents while imposing certain conditions, such as attributing the source of the data. Unlike the CC0 license, which places a work in the public domain, the DBCL 1.0 maintains some restrictions to ensure proper attribution and integrity of the database contents. It strikes a balance between openness and the requirement to acknowledge the source of the data when used in derivative works. Because the DBCL 1.0 is more restrictive, it is a suitable choice for our dataset.
