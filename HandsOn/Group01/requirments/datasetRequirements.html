Parking History Analytics Dataset Requirements

This document aims to assess if our dataset meets the specific requirements needed for our project. The following checklist provides a summary of each requirement, and a brief description of how our dataset satisfies them.

  • R1. Data are in the smart cities domain: Sensors in parking bays that provide real-time data on parking availability are a prime example of how digital technology can improve smart cities, as they can help manage traffic flow, reduce congestion, and improve urban planning.

  • R2. Data are available as a CSV file: The data has been downloaded in a CSV format from The City of Melbourne Open Data Website.

  • R3. Data have an open license so it can be published: Their license is CC BY 4.0 LEGAL CODE (Creative Commons).

  • R4. Data can be easily linked with generic real-world entities (e.g., locations): The Location column provides spatial coordinates that correspond to real-world locations.

  • R5. Documentation exists for the data (optional): Further information about City of Melbourne's parking sensors can be found here.

  • R6. Data can come from multiple data sources (optional): There are three datasets that make up the live parking sensor release. They are the on-street parking bay sensors, on-street parking bays and the on-street car park bay restrictions.
