Waste Management Analytics Dashboard Requirements

This application requirements for the Waste Management Analytics Dashboard based on the waste collection data from the city of Madrid.

Functional Requirements:

  1. Display waste collection data in monthly/yearly intervals.
  2. Provide visual analytics of waste production by district.
  3. Showcase trends such as districts that have reduced waste over time or emerging waste categories.
  4. Offer a detailed breakdown of waste types, either for individual districts or for the city overall.

Mock-ups:

Dashboard Main View:

An intuitive layout displaying an overview of waste collection in monthly/yearly intervals. Users can quickly switch between views. Interactive charts show total waste production, and a sidebar allows for district-specific filtering.

District-specific Waste Analysis:

When a district is selected, the dashboard transitions to show data specific to that district. Pie charts or bar graphs visualize the breakdown of waste types. A line graph could represent the waste trends over time.

Waste Type Breakdown:

This section gives users a more in-depth look into specific waste categories. Through dropdown menus or tabs, users can select a waste type and view its production trends, both citywide and district-specific.
