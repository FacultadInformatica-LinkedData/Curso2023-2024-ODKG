Parking History Analytics Dashboard Requirements

This application requirements for the Parking History Analytics Dashboard based on the "On-street Parking Bay Sensors"

Functional Requirements:

  1. Display parking patterns over various time frames, such as daily, weekly, monthly, and annually.
  2. Show graphs/charts for:
    • Peak parking times
    • Most frequently occupied spots
    • Zone-wise occupancy trends
  3. Allow data filtering by "Zone_Number" to enable more granular insights.
  4. Provide a user-friendly date range filter for custom date analysis.

Mock-ups:

Dashboard Main View:

A clean layout with a navigation bar on the top with options such as "Daily", "Weekly", "Monthly", "Annual", and "Custom" views. Below this, a date range picker allows users to select custom date ranges. The main content area displays interactive charts showing parking data patterns, and on the side, a panel allows users to select specific "Zone_Number" for focused analysis.

Zone-Specific Analysis:

Upon selecting a specific "Zone_Number", the charts and graphs update to reflect data specific to that zone. Below the main charts, a list of the most frequently occupied spots within the zone is displayed, potentially with links to their exact locations or additional details.
