Data analysis

We have selected 5 different CSV files regarding measurements and the environment in the city of Barcelona. Each one will be analysed individually.

Temperaturesbarcelonadesde1780.csv

  • There are 243 rows in the CSV file. Each one is complete with no missing data.
  • The schema for this data is very simple. Each row is self-contained and has average temperature measurements for each month of that year.
  • The first column is called ‘Any’ but it refers to the years for the measurements. Every integer from 1780 to 2022 (inclusive) is present in ascending order.
  • The remaining columns are numeric, float values recording the average temperature for the year.

qualitat_aire_contaminants.csv

  • There are 3 columns and 21 rows in this CSV file. It is a small CSV with a very simple schema, just listing the code, description and units for each row of contaminant.
  • The first column is “Codi_Contaminant”, storing an integer code to identify each contaminant. These values are all unique and have the range 1 - 999.
  • The second column is “Desc_Contaminant”, containing a string description of the contaminant. This is usually its chemical symbol e.g. CO but occasionally consists of words e.g. “Black Carbon”. 
  • The third column is “Unitats”, storing the units for each measurement. There are 3 units used here, µg/m³ (the most common unit with 15 occurrences), mg/m³ and %. There are 3 rows about flow which are missing a unit.

OD_Arbrat_Zona_BCN.csv

  • This is a large CSV file with 42944 rows describing trees in Barcelona.
  • The first column is “codi” with a unique 9 character string identifier.
  • The second and third columns are ‘x_etrs89’ and ‘y_etrs89’
  • The fourth and fifth columns are ‘latitud’ and ‘longitud’, storing the latitude and longitude of the tree.
  • The sixth column is “tipus_element” where there are only 2 possible string categories: “ARBRE_ZONA” (the most common) and “PALMERA_ZONA” for tree and palm tree zones respectively.
  • The seventh column is “espai_verde”, the green space where the tree is located in textual format. These can be duplicated.
  • The eighth column is “adreca”, the address of where the tree is in textual format. A few values here are incomplete such as a number with a comma.
  • The ninth column is “cat_especie_id”, a numeric identifier for the species of tree and the corresponding scientific name is in the tenth column “nom_cientific” as a string. Likewise, the corresponding popular names in Spanish and Catalan are in the eleventh and twelfth columns respectively (“nom_castella” and “nom_catala”) as a string.
  • The thirteenth column is “categoria_arbrat” with a tree category: “PRIMERA”, “SEGONA” or “TERCERA”.
  • The fourteenth column is “data_plantacio” with the date at which the tree was planted in the string format “YYYY-MM-DD”. This column has many missing entries. 
  • The fifteenth column is “tipus_aigua” with the type of water: “FREATICA”, “POTABLE”, “REGENERADA”. This column has mostly missing entries.
  • The sixteenth column is “tipus_reg”, storing the category for the irrigation type. There are 8 text choices for this and a few entries are missing values for this column.
  • The seventeenth column is “geom”, representing the location of the tree in the POINT format.
  • The eighteenth column is “catalogocio”: it either has the value “INTERES LOCAL” or is missing a value.
  • The nineteenth column is “codi_barri” with an integer identifier for the neighbourhood, accompanied by the name in text in the twentieth column “nom_barri”.
  • The twenty first column is “codi_districte” with an integer identifier for the district, accompanied by the name in text in the twenty second and last column “nom_districte”.

2023_qualitat_aire_estacions.csv

  • This is relatively small for a CSV with 54 rows tracking air station measurements.
  • The first column “Estacio” records an integer number in the range 4 - 59 which identifies the station. The second column “nom_cabina” is the neighbourhood where the station is positioned, stored as a string. The third column “codi_dtes” records a 2 letter string code for the station, with IZ as the most common one with 21 occurrences. There are 7 distinct values here.
  • The fourth column is “zqa” where every row has the integer value 1.
  • The fifth column is “codi_eoi”, a 7 digit number. This corresponds to the address in the eighth column “ubicacio”.
  • The sixth and seventh columns are floats for “Longitud” and “Latitud”, storing the longitude and latitude to 4 decimal places.
  • The ninth column is “Code_districte”, storing an integer code for the district and the corresponding name is in the tenth column “Nom_districte”. These codes are in the range 1 - 10.
  • The eleventh column is “Code_barri”, storing an integer code for the area and the corresponding name is in the twelfth column “Nom_barri”. These codes are in the range 4- 69.
  • The thirteenth column is “Clas_1”, identifying the type of area between “Suburbana” and “Urbana”, suburban and urban areas.
  • The fourteenth column is “Clas_2”, identifying background and traffic respectively with the 2 options, “Fons” and “Trànsit”.
  • The fifteenth and last column is “Codi_Contaminant”. This stores an identifier for the contaminant which can be linked to the column of the same name in qualitat_aire_contaminants.csv.

2022_tramer_pm10_mapa_qualitat_aire_bcn_csv

  • This CSV has 15041 rows and 3 columns.
  • The first column is “TRAM”, a unique string identifier for a street section code. It is a 7 character string, starting with a “T”, followed by 5 numbers and a letter such as “T04719W”.
  • The second column is “Rang”, storing the range of the air quality measurement. There are 6 categories: 15-20, 20-25, 25-30, 30-35, 35-40 and >40µg/m³. 
  • The third column is GEOM_WKT, containing a 4-tuple of locational information in a MultiLine. This is a “Well-Known Text” format which represents the two end points of a line, i.e. a street, with the X and Y coordinates for both endpoints as the 4 numbers.
